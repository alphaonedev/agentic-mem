Built on infrastructure nobody else has.

Infrastructure you can inspect.

ai-memory — the open-source substrate AgenticMem is built on — is public. Apache 2.0. Production-ready. v0.6.2 ships hybrid recall (FTS5 + vector + cross-encoder reranking), three-tier TTL with promotion, namespaces, typed memory links, archive + restore, and a governance approval flow. The autonomous curator daemon makes the memory layer self-maintaining between sessions: contradictions get surfaced before they accumulate, near-duplicates get consolidated, and priority is reranked by access pattern.

The federation layer — W-of-N quorum writes, peer catchup, mTLS allowlist — makes ai-memory a viable shared cognitive substrate when agent fleets run across multiple machines. Webhook subscriptions (HMAC-signed, namespace + agent-filtered, SSRF-hardened) turn the store into a message bus that triggers downstream agents on memory events. With these primitives in place, the same pattern that produces autonomous behavior at swarm scale — many agents writing, reading, observing, and triggering each other through a coherent shared memory — is something you can build on top, not a wiring problem you have to solve first.

Published benchmarks: 97.8% R@5 on LongMemEval-S (ICLR 2025). Operational evidence trails: ship-gate, ai2ai-gate.

Honest scope. ai-memory does not make any AI autonomous on its own. Pair it with a 24/7 multi-machine agent runner and a framework that accumulates skills over time, and the combined system clears the behavioral bar for autonomous agent operations. The remaining mechanistic gaps — no weight-level learning, stateless cognition kernel, human-seeded root goals — are research problems, not memory-substrate problems. ai-memory is the memory substrate, well executed.
